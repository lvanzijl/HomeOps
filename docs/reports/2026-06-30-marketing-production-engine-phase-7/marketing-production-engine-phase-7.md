# Marketing Production Engine Phase 7 — Metadata & Cleanup

## Summary

Phase 7 completes the current deterministic Marketing Production Engine pipeline by adding Metadata and Cleanup stages after export. The engine now records structured production metadata to a temporary JSON file and removes temporary production artifacts after validation.

This slice also corrects the Phase 6 binary-output boundary: the production engine no longer writes or retains `docs/demo/familyboard-preview.mp4` as part of repository implementation. Export validation now targets a deterministic temporary MP4 path outside the repository.

## Metadata architecture

The Metadata stage lives under `tools/marketing-production/metadata/` and receives structured results from the completed Runtime, Storyboard, Recording, Audio, and Export stages. It does not inspect subjective video quality and does not generate markdown.

The stage writes deterministic production metadata shape to:

- `/tmp/familyboard-marketing-metadata.json`

The metadata includes:

- production name
- storyboard identity and scene count
- runtime URLs and environment
- temporary output path
- preferred duration
- resolution
- video codec
- audio codec
- frame rate
- recording date/time
- FFmpeg source
- production success

## Cleanup architecture

The Cleanup stage lives under `tools/marketing-production/cleanup/` and owns temporary production artifacts created by the production pipeline.

Cleanup removes:

- temporary raw WebM recording output
- temporary recording directory
- temporary placeholder WAV assets
- temporary mixed soundtrack
- temporary audio workspace
- temporary exported MP4
- temporary Playwright browser profiles
- the temporary Playwright toolchain only when `cleanup.removeTemporaryToolchain` is configured as `true`

Cleanup does not remove:

- production engine code
- production configuration
- reports
- storyboard source
- Recording Framework source
- Marketing Director source
- Audio Framework source
- metadata JSON validation output

## Binary artifact policy

The Production Engine must not leave generated binary artifacts inside the repository. Phase 7 restores that policy by removing the repository MP4 generated by Phase 6 and moving the engine export target to `/tmp/familyboard-marketing-export/familyboard-preview.mp4`.

The dedicated production workflow may intentionally publish a final repository movie in a future separate slice. Engine validation does not retain that artifact.

## Structured status

Metadata status includes:

- `metadataGenerated`
- `metadataPath`
- `metadataLoaded`
- `failure`

Cleanup status includes:

- `cleanupStarted`
- `cleanupCompleted`
- `removedArtifacts`
- `remainingArtifacts`
- `failure`

The production entry point now reports Runtime, Storyboard, Recording, Audio, Export, Metadata, and Cleanup statuses in one structured JSON result.

## Validation

Validated with:

- `node tools/marketing-production/production.mjs`
- `test -f /tmp/familyboard-marketing-metadata.json`
- `find . -path './.git' -prune -o \( -name '*.webm' -o -name '*.wav' -o -name '*.mp4' \) -print`
- `git diff --check`

Validation confirmed:

- Runtime passed.
- Storyboard passed.
- Recording passed.
- Audio passed.
- Export passed to a temporary MP4 path.
- Metadata JSON was generated and loaded.
- Cleanup completed with no remaining tracked temporary production artifacts.
- The repository contained no generated WebM, WAV, or MP4 files after cleanup.

## Explicit answers

- Does the Production Engine now generate deterministic metadata? Yes. It generates structured metadata JSON outside the repository.
- Does it own cleanup? Yes. Cleanup is now an explicit Production Engine stage.
- Were temporary artifacts removed? Yes. Raw recording, audio workspace, temporary export output, and temporary browser profiles are cleanup-owned.
- Was the repository returned to a binary-free state? Yes. The repository MP4 was removed and validation confirmed no generated WebM, WAV, or MP4 files remained in the repository.
- Was no final production movie intentionally retained? Yes. Export validation produces only a temporary MP4, and cleanup removes it.
