# Child Workspace Overview First Implementation

## Summary
- Reordered Child Mode so Today appears immediately after identity and answers what to do now before progress storytelling.
- Compacted the primary overview into Today, one progress signal, compact Family Goal context, and a latest Helpful Moments preview.
- Moved full personal progress, Family Goal detail, celebration memories, and appreciation history behind explicit detail access.
- Added a small Parent Mode discovery cue while preserving the separate Parent Mode tab and keeping parent controls out of Child Mode.

- A child can answer “What should I do today?” from the first Child Mode card.
- A child can answer “How am I doing?” from the single primary progress summary.
- A child can answer “What are we working toward together?” from the compact Family Goal summary.
- Deeper exploration still exists through explicit progress, family-goal, memory, and appreciation controls.
- Cross-page UX rule check: Child Workspace now prioritizes overview over detail by defaulting history, expanded progress, and management concepts to deliberate access.
